# `docs/strata` — Artifact Trail Reference

This folder illustrates the governance artifact structure inside a project driven by the [STRATA Protocol](https://strataprotocol.org).

The files here are drawn from the real `strataprotocol.org` project and are provided as concrete examples of what each artifact looks like in practice. They are not templates — they are the actual documents that governed the development of this platform.

## Folder structure

docs/strata/
├── 1.classification/       # Stratum 1 — Classification Record
├── 2.derivation/           # Stratum 2 — Vision, BRD, PRD, Engineering Standards
├── 3.authority-chain/      # Stratum 3 — Copilot instructions and authority chain
├── 4.execution-loop/       # Stratum 4 — Session folders (analysis, implementation plans, manifests)
│   └── sessions/
│       └── REQ-NNN-*/      # One folder per feature requirement
└── 5.artifact-trail/       # Stratum 5 — Artifact trail index and completion records


## What's included

Populated files are actual artifacts from the `strataprotocol.org` project — real governance documents produced during real development sessions. They show how the five strata of the STRATA Protocol produce a living, traceable record of every decision from classification through delivery.

Empty folders contain a `.gitkeep` placeholder. They exist to illustrate the complete structure a governed project maintains, even when the corresponding artifacts are not included in this public illustration.
